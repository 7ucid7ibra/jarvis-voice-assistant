üß¨ Tech Specs: Model Configuration ‚ñº

Jarvis uses a unified provider interface. You can switch providers instantly in settings:

  • Local: Uses `subprocess` to talk to a local Ollama instance (default port 11434). Supports model catalog search and pull.
  • Cloud: Standard HTTP requests to OpenAI/Gemini/OpenCode APIs. Supports JSON Schema output for structured actions.
  • Routing: The OpenCode provider automatically handles the difference between OpenAI-style and Anthropic-style message payloads.

üè† Integration Details ‚ñº

Dynamic Context Injection: Before every request, Jarvis fetches the latest state of your devices and injects it into the system prompt:

// Context injected into LLM:
- light.studio_main: state=on, brightness=255
- sensor.temperature: state=21.5
- input_boolean.guest_mode: state=off

// Model Output (Action Agent):
{
  "service": "light.turn_off",
  "entity_id": "light.studio_main"
}

This ensures Jarvis never hallucinates device states.

Get Started

  1. Download the source code from GitHub.
  2. Open your terminal and navigate to the project folder.
  3. Run the installer command:
$ ./scripts/start.sh

This script automatically configures the environment and launches your new assistant.
